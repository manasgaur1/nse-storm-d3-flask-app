Analytics on Real Time Stock Data

The Above Graphic is Not Active between 07:00 to 10:00 AM Indian Standard Time Quotes from the NSE Website with a time delay of 5-15 minutes Best Viewed on a Laptop Browser. Hover over a Stock Symbol to Check it's Traded Volume

This Proof Of Concept (POC) Demonstrates how Stock Exchange Data can be Analyzed in Real Time. The Graphic above displays Stocks that have Share Trade Volumes more than 5,000,000 on India's National Stock Exchange. The following Phases of this Concept have all been Implemented with Open Source Technologies:

  • Data Sourcing
  • Data Parsing and Analytics
  • Data Visualization

Data Sourcing

For POC purposes, we were expecting Stock Quote APIs to be easily available; possibly like Google Maps APIs. Wishful thinking! Most Stock Exchanges don't provide Real Time Quotes for Free. After some research, we found an awesome python library Nsetools that parses the NSE Website and returns a set of key-value pairs for each Stock. This fulfilled our needs for the POC and we didn't look further. However, for a Production Implementation it would be best to get the feed from the Stock Exchanges or from a reputed Subscription API such as BLPAPI.

A few keys from the API that can be used to create Metrics:

deliveryQuantity
deliveryToTradedQuantity
averagePrice
dayHigh

Data Parsing and Analytics

We decided to use Apache Storm for Parsing and running Anaytics. It is an Open Source Computation System that works on Real Time Streams on a distributed environment. It was originally developed to perform Real Time and Historical Analysis of the Twitter Firehose. You can imagine it's Capabilities considering that it is widely used to Analyze Real Time Tweets. The Software has been hosted on Amazon Web Services

Data Visualization

We have used D3.js for Visualizing the Stock Symbols with Traded Quantity more than 5,000,000. If you hover over a Symbol, it will display the Traded Quantity Value as a tooltip. D3.js has the capabilities of producing dynamic, interactive data visualizations. All sorts of Charts & Graphs can be produced. Redis acts as a Message Broker between Storm and the Web-Server.

What Next?

The idea is to allow Users to create their own Metrics from the Front End, instead of pre-defined ones (such as Traded Volume > 5000000). Keep Watching this space!!
